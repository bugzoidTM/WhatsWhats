🔑 Provider de IA

O bot responde primeiro pelos fluxos. Se nenhum fluxo bater, ele usa o provider de IA abaixo. Além do Groq, você pode usar qualquer API compatível com OpenAI informando chave, Base URL e modelo.

Para provider custom, informe um endpoint compatível com /chat/completions.

📋 Fluxos do Bot

Quando o cliente digitar uma das palavras-chave, o bot responde automaticamente. Separe palavras por vírgula (ex: oi, olá, bom dia).

⚙️ Prompt e Modelo da IA

Substitua [SUA EMPRESA] pelo nome real. A IA usa este texto para saber como responder.

Use 160–220 para WhatsApp. Isso reduz respostas longas e confusas.

🔗 Webhooks

Configure URLs para receber eventos da instância em tempo real. Compatível com o padrão Evolution API. Cada evento envia um POST com o payload JSON para a URL configurada.

➕ Adicionar Webhook

📖 Referência dos Eventos

messages.upsert Mensagem recebida ou enviada pelo bot
connection.update WhatsApp conectou ou desconectou (state: "open" | "close")
qrcode.updated Novo QR Code gerado (base64 e raw)
messages.read Mensagem lida pelo destinatário
presence.update Status de digitação / presença (futuro)
custom Evento personalizado — envie pelo endpoint /api/:instance/send
Payload padrão: 
{
  "event": "messages.upsert",
  "instance": "nome-da-instancia",
  "data": { ... },
  "timestamp": 1713983200000,
  "date_time": "2025-04-24T20:00:00.000Z"
}
